Step 1 of 4 backend

Diff span similarity gating and token snapping

Prevent misleading intra-line change marks on unrelated/rewritten lines and keep marks aligned to whole tokens.

ChangedSpans now measures non-whitespace similarity between the shared prefix/suffix and the shorter line's content; below a 50% threshold it treats the lines as an unrelated rewrite and suppresses marks entirely. When marks are kept, prefix/suffix boundaries are widened to word edges so a mark never starts or ends mid-identifier. A related fix in highlight() skips rendering an empty <mark> for zero-width spans, and the chroma theme builder now suppresses the Error token's inverse color scheme since isolated punctuation frequently lexes as Error and became invisible once backgrounds were stripped.

⚠ Worth double-checking

  • Confirm the 0.5 similarity threshold matches the intended GitHub-like behavior and doesn't over-suppress legitimate short edits.
  • Check word-rune widening logic for off-by-one errors near string boundaries (prefix==0, suffix==0 cases).
